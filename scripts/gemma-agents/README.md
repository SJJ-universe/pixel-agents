# Gemma agents → pixel-agents office

pixel-agents only _visualizes_ agent activity — it never calls an LLM. This runner
is the agent side: it runs N Gemma-powered loops (via OpenRouter) and streams the
same events the office understands, so N characters type/read while Gemma "works"
and wander around between turns. The OpenRouter key stays in Node — never in the browser.

```
runner.mjs (Node, OpenRouter) --SSE 127.0.0.1:7777--> webview dev client --window 'message'--> 7 characters
```

The activity labels ("로그인 폼 고치는 중") are real Gemma output; the agents don't
perform real code edits. It's a live, LLM-driven visual demo of the office.

## Run — env file is the only thing you fill in

1. Set your key (model already defaults to lightweight Gemma 4):

   ```
   cp scripts/gemma-agents/.env.example scripts/gemma-agents/.env
   # edit .env: OPENROUTER_API_KEY=sk-or-...
   ```

2. From the repo root, start the runner **and** the webview dev server together:

   ```
   npm run gemma:dev
   ```

   (or run them separately: `npm run gemma` and, in another terminal, `npm run dev:webview`)

3. Open **http://localhost:5173** and toggle **3D**. Seven Gemma agents appear, each
   showing a Gemma-generated activity (typing/reading) and wandering between turns.

Stop with Ctrl+C (the agents despawn).

## Model

Default `GEMMA_MODEL=google/gemma-4-26b-a4b-it` — lightweight Gemma 4 (26B total but
only ~4B active params, so fast and ~$0.06/Mtok in; calls are ~24 tokens each, so cost
is pennies for hours). Set `google/gemma-4-26b-a4b-it:free` for zero cost (the free
tier may rate-limit with 7 agents — the runner falls back to canned labels per call when
that happens, so it never breaks).

## Real code editing — `coder.mjs`

`runner.mjs` only generates activity _labels_. `coder.mjs` is the real thing: each Gemma
agent gets actual tools (`read_file` / `list_dir` / `write_file` / `run_cmd` / `finish`),
**all jailed to one workspace folder**, and runs an OpenAI tool-calling loop — so the
agents genuinely build software while the office visualizes the real tool calls
(write→typing, read→reading, run→bash).

Flow: 총괄 plans (`submit_plan` → file-scoped subtasks) → 아키텍트/백엔드/프론트엔드
implement in parallel on disjoint files → 테스터 runs tests & fixes → 총괄 writes `SUMMARY.md`.

```
# preview the plan only (one cheap call, no writes):
node scripts/gemma-agents/coder.mjs --dry-plan

# full run + office:
TASK="build a Node CLI todo app" npm run coder:dev
# then open localhost:5173, toggle 3D
```

Key source order: `OPENROUTER_API_KEY` env → `scripts/gemma-agents/.env` →
`~/Desktop/openR.env` (accepts a bare key). Config via env:

- `TASK` — what to build (default: a small Node todo CLI)
- `WORKSPACE_DIR` — sandbox folder the agents edit (default `~/Desktop/gemma-workspace`)
- `ALLOW_RUN=0` — disable `run_cmd` (read/write only, no shell exec)
- `MAX_STEPS` (14), `RUN_TIMEOUT_MS` (60000)

Safety: every file/dir op is path-jailed to `WORKSPACE_DIR` (escapes rejected); `run_cmd`
runs with `cwd=WORKSPACE`, a timeout, and a deny-list for catastrophic commands. It still
executes real shell commands the model chooses — point it at a throwaway workspace, not a
repo you care about. Self-check (no API): `node scripts/gemma-agents/coder.mjs --selftest`.

Caveat: lightweight Gemma 4 is far weaker than Claude at multi-step coding — expect rough
edits, occasional dead-ends, and bounded results. The harness caps steps and falls back
gracefully, but it's a real (if modest) coding swarm, not a Claude-Code replacement.

## Notes

- No key set → the runner still animates with canned activities (so you can see it work),
  and prints a warning. With a key, the activity labels are generated by Gemma in real time.
- `AGENTS` (default 7) and `PORT` (default 7777) are configurable in `.env`.
- The bridge is **dev-only** (`webview-ui/src/devGemmaBridge.ts`, gated on `import.meta.env.DEV`);
  it's never in a production build. For a production/standalone setup, the runner would
  POST hook events to `npx pixel-agents`'s `/api/hooks/claude` instead (Bearer token from
  `~/.pixel-agents/server.json`) — same event shapes.
